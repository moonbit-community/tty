# Color And Output Commands

## Goal

Move semantic color API out of `vt/color`, keep `vt` focused on low-level VT
protocol bytes, and add command-style `Output` helpers for common screen,
cursor, and SGR operations.

## Status

Done.

## Context And Decisions

- `vt` is the protocol layer. It should expose byte constructors for CSI, SGR,
  DEC private modes, scroll margins, and related terminal sequences.
- `Color` and `BasicColor` are user semantic values, not VT protocol concepts.
  They belong in `tonyfettes/tty/color`.
- `vt/color` remains as a compatibility re-export for the existing public API.
  New demos and root APIs should import `tonyfettes/tty/color`.
- Top-level `Output` methods provide the command-style user API. They write the
  bytes generated by `vt` and `color` to the output stream.
- `reset_style` means SGR 0. It resets foreground, background, and all text SGR
  attributes. It is intentionally distinct from `reset_foreground` and
  `reset_background`.
- `Output::with_alt_screen` is allowed as a simple scoped helper for now.
  Larger guard composition can be addressed later by a terminal session API.

## Public API Changes

Add `tonyfettes/tty/color` with:

- `BasicColor`
- `Color`
- `foreground(Color) -> Bytes`
- `background(Color) -> Bytes`
- `reset_style : Bytes`
- `reset_foreground : Bytes`
- `reset_background : Bytes`

Add `tonyfettes/tty/vt` low-level SGR helpers:

- `sgr1(Int) -> Bytes`
- `sgr3(Int, Int, Int) -> Bytes`
- `sgr5(Int, Int, Int, Int, Int) -> Bytes`

Add root `Output` methods:

- `enter_alt_screen()`
- `leave_alt_screen()`
- `with_alt_screen(async () -> T) -> T`
- `hide_cursor()`
- `show_cursor()`
- `set_foreground(@color.Color)`
- `set_background(@color.Color)`
- `reset_foreground()`
- `reset_background()`
- `reset_style()`

Keep `tonyfettes/tty/vt/color` as a compatibility re-export for its existing
public API.

## Invariants

- `vt` remains byte-only and does not import root `tty` or write to streams.
- `color` remains byte-only and does not import root `tty` or read environment
  variables.
- Root `Output` methods do not track terminal state. They only write command
  sequences to the output stream.
- No color capability detection is added in this task.

## Acceptance Criteria

## Public API Audit

- Added `tonyfettes/tty/color` as the semantic color package. It exposes
  `BasicColor`, `Color`, `foreground`, `background`, `reset_style`,
  `reset_foreground`, and `reset_background`.
- Added low-level `tonyfettes/tty/vt.sgr1`, `sgr3`, and `sgr5`.
- Added root `Output` command helpers for alternate screen, cursor visibility,
  foreground/background colors, and SGR style resets.
- Kept `tonyfettes/tty/vt/color` as a compatibility package. Its public
  `Color` and `BasicColor` names now re-export the semantic `color/` types, and
  `reset_all` remains only as a compatibility alias.
- No color detection, terminal state tracking, or screen rendering API was
  added.
