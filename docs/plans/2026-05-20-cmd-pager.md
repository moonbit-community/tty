# Pager Demo

## Goal

Add a `cmd/pager` manual validation demo for paging text with a fixed status
line. The demo should exercise raw input, terminal size querying, top/bottom
margins, and Reverse Index without introducing a public pager or line-editor
API.

## Status

Done.

## Context And Decisions

- This is a demo under `cmd/*`, not a library package.
- The pager uses `moonbitlang/core/argparse` to read one required file path from
  argv. Running without a file is an argparse error.
- It uses the primary screen rather than alternate screen so output can remain
  in terminal scrollback after exit.
- Short content is rendered inline from the current cursor position, like
  `more`, without moving to row 1 or clearing existing terminal content.
- Long content first writes a full viewport naturally so older terminal content
  moves into scrollback, then reserves the bottom row for status and sets the
  scrolling margins to the content rows above it.
- Single-line upward movement uses RI at the top scrolling margin.
- Normal exit preserves the final status line and leaves the shell prompt on the
  following line.
- There is no resize event yet; each input refresh re-queries terminal size.

## Public API Changes

None. `cmd/pager` is a demo package.

## Invariants

- No root, `input`, or `vt` public API is added for pager behavior.
- The demo does not own reusable scrollback, layout, or widget abstractions.
- Terminal margins are reset and cursor visibility is restored on normal exit
  and on raised errors.

## Acceptance Criteria

- `moon run cmd/pager -- <file>` starts a pager over file content.
- `moon run cmd/pager` reports that the required file argument is missing.
- `moon run cmd/pager -- --help` prints parser-generated help.
- `q` and `Ctrl-C` exit.
- `j`/Down and `k`/Up scroll by one line.
- Space/PageDown and `b`/PageUp scroll by a page.
- Status stays on the bottom row while content rows scroll.

## Validation Commands
