# VT Color Sequences

## Goal

Add a `tonyfettes/tty/vt/color` package for constructing SGR color byte
sequences, plus a `cmd/color` manual specimen demo.

## Status

Done.

Follow-up: `tonyfettes/tty/vt/color` and `tonyfettes/tty/vt/internal/seq` were
later removed by `docs/plans/2026-05-20-vt-package-cleanup.md`. The current
semantic color API lives in `tonyfettes/tty/color`, while `vt` keeps private CSI
helpers and public low-level SGR helpers.

## Context And Decisions

- `vt/color` is a child of the existing pure `vt` byte-sequence boundary.
- `vt/color` only emits bytes. It does not read environment variables, query
  tty handles, detect terminal capabilities, or decide whether colors should be
  enabled.
- Shared CSI/SGR byte construction belongs in `tonyfettes/tty/vt/internal/seq`,
  not in a public `vt/common` package. The `internal/` package keeps helper
  names available to `vt` and `vt/*` descendants without making them a general
  public API.
- Color capability detection is intentionally left for a future top-level
  `tonyfettes/tty/color` package or plan.
- Reset names are scoped to `vt/color` and must be explicit:
  `reset_all`, `reset_foreground`, and `reset_background`.
- `cmd/color` is a demo and specimen renderer, not public library API.

## Public API Changes

Add package `tonyfettes/tty/vt/color` with:

- `BasicColor`
- `Color`
- `foreground(Color) -> Bytes`
- `background(Color) -> Bytes`
- `reset_all : Bytes`
- `reset_foreground : Bytes`
- `reset_background : Bytes`

`BasicColor` and `Color` are intentionally constructible by callers because the
package is a value-to-sequence emitter. Invalid numeric channels are clamped at
the sequence boundary.

## Invariants

- `vt/color` has no dependency on the root `tty` package.
- `vt/color` has no environment, terminfo, or platform FFI dependency.
- `vt/color` returns `Bytes` and never writes to an output stream.
- `vt/internal/seq` exposes only low-level sequence builders needed by `vt` and
  `vt/color`.
- The existing `tonyfettes/tty/vt` package API is not widened just to re-export
  color helpers.

## Acceptance Criteria

- ANSI 8/16 foreground and background colors are emitted as SGR sequences.
- Indexed 256-color foreground and background sequences are emitted with
  clamped `0..255` indices.
- Truecolor RGB foreground and background sequences are emitted with clamped
  `0..255` channels.
- Foreground, background, and all-attribute resets are available.
- `cmd/color` prints visible specimens for basic colors, 256-color indices, and
  truecolor gradients.

## Public API Audit

- `tonyfettes/tty/vt/color` exposes only `BasicColor`, `Color`,
  `foreground`, `background`, `reset_all`, `reset_foreground`, and
  `reset_background`.
- `BasicColor` and `Color` are constructible by design because callers need to
  pass color values directly to the emitter functions.
- `tonyfettes/tty/vt/internal/seq` exposes CSI/SGR builders only through an
  `internal/` package, so they are importable by `vt` descendants without
  becoming general user-facing API.
- `tonyfettes/tty/vt` was not widened to re-export color helpers.
- `cmd/color/pkg.generated.mbti` exposes no values or types.
