# Agent Demo

## Goal

Add a neutral `cmd/agent` primary-screen demo that combines transcript output
with a trailing input region. The demo should validate the building blocks
needed for a Codex-like terminal experience without introducing a public TUI,
renderer, or agent API.

## Status

Done.

## Context And Decisions

- `cmd/agent` is a demo under `cmd/*`, not a library package.
- The transcript stays in the terminal primary screen and should enter normal
  terminal scrollback.
- The input region is rendered immediately after the transcript, like `more`
  style primary-screen output. The demo does not jump to the terminal bottom or
  fill the screen with blank lines at startup.
- New transcript rows overwrite the current input region, then the input region
  is redrawn below them. Once output reaches the terminal bottom, this advances
  the primary screen naturally and pushes transcript content into scrollback.
- The terminal cursor stays visible and is repositioned to the editor cursor
  after each redraw.
- The demo uses existing input decoding, UTF-8 text events, grapheme-aware
  backspace, terminal size querying, and `vt/color` for small visual accents.
- `Enter` submits the input. `Ctrl-J` inserts a newline into the input buffer;
  the trailing input region shows the last two input lines.
- The input composer owns a demo-local minimal editor state: buffer text,
  grapheme-indexed cursor, preferred display column for vertical movement, and
  the first visible composer line. Up and Down move the cursor between logical
  lines; the visible composer window follows the cursor when the buffer has
  more lines than the visible composer rows.
- Assistant responses are deterministic fake responses. This demo does not call
  a model, run tools, stream tokens, render markdown, or manage conversation
  state beyond a short in-memory transcript.
- There is no resize event yet. The demo re-queries terminal size in the input
  loop and redraws the input region with the current size.

## Public API Changes

None. `cmd/agent` is a demo package.

## Invariants

- No root, `input`, `vt`, or `vt/color` public API is added for agent behavior.
- `cmd/agent` must not switch to the alternate screen, clear the whole
  terminal, or jump to the bottom of the viewport on startup.
- Terminal margins are reset, cursor visibility is restored, and SGR attributes
  are reset on normal exit and on raised errors.
- During normal operation, the terminal cursor remains visible in the input
  region.
- The input buffer remains demo-local; it is not promoted to public line-editing
  API.
- Viewport position is derived from editor cursor visibility. There is no
  independent composer scroll offset that can disagree with the insertion
  point.

## Acceptance Criteria

- `moon run cmd/agent` starts a primary-screen demo at the current cursor
  location without filling the viewport with blank lines.
- Typing text appends Unicode text to the input buffer.
- Backspace deletes one grapheme cluster.
- Ctrl-J inserts a newline into the input buffer.
- Enter submits the current input as a user message and appends a fake
  assistant response above the input region.
- The cursor remains visible at the editor insertion point.
- Up and Down move the editor cursor across input lines and scroll the visible
  composer only as needed to keep that cursor visible, without changing
  transcript scrollback.
- Ctrl-C exits while resetting margins, color, and cursor visibility.
